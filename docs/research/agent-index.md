# Research Agent Index

Read this file before opening other files under [`docs/research/`](./).

## Purpose

- This directory holds research briefs that support tasks, decisions, or follow-up work.
- Keep source lists, comparisons, findings, and evaluation checklists here so task files stay focused on scope, recommendations, and outcomes.

## Layout

- Keep the folder flat for now and store briefs as [`docs/research/<topic>.md`](./).
- Prefer stable topic-based filenames so more than one task can link to the same research brief.
- Research briefs do not need a `Status` section by default.
- If the directory grows too large later, group files by topic rather than by workflow status.

## Load Rules

- Open only the research brief that matters for the current task or decision.
