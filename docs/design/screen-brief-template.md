# Screen Brief Template

Use this file as the starting point for any task-specific Penpot brief under [`docs/design/`](./).

## Artifact

- Repo artifact directory: [`./`](./)
- Repo SVG export: `name-your-artifact.svg` (replace with the committed SVG filename in this folder)
- Optional Penpot share link: `not required for the repo workflow`
- Status: `draft`

## Goal

- Describe the screen or flow this brief owns and what changed since the last artifact.

## Screens

- List the screen, frame, or flow this brief covers.
- Start from the mobile-first baseline in [`foundations/README.md`](foundations/README.md) unless the task is explicitly changing the foundation itself.

## Layout and Spacing Notes

- Describe the smallest supported layout first.
- Note when tablet or wider layouts should diverge and why.

## Key States or Variants

- List empty, loading, error, active, expanded, confirmation, or other meaningful states.

## Content and Copy Notes

- Record the exact labels, helper copy, and sample values that matter for implementation or review.

## Implementation Notes for Codex

- Call out reusable components, expected hierarchy, accessibility constraints, and any layer-naming expectations for Penpot.

## Open Questions or Follow-Up

- Capture unresolved details here instead of leaving them only in chat memory.
