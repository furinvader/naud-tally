# T-008: Add i18n Support

## Status

- open

## Goal

- Make the UI support multiple languages, starting from the current English-first structure.

## Done when

- user-facing strings are organized behind an i18n-ready structure
- English remains the default language
- the current feature structure no longer blocks adding another locale
- the chosen i18n approach is recorded in the docs

## In scope

- string externalization
- initial translation structure
- documentation of the chosen i18n direction

## Out of scope

- remote translation management
- machine-generated translations
- non-text localization work beyond what the chosen approach requires
