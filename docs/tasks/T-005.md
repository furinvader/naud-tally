# T-005: Build the Guest Tally Screen

## Status

- done

## Goal

- Let guests quickly record drinks on a tablet.

## Done when

- the fixed sample drink list is shown as touch-friendly cards
- each drink supports increment and decrement controls with visible counts
- the total drink count stays visible while using the screen
- tally state persists across reloads on the same device
- unit tests cover the main UI flow and tally store behavior

## In scope

- the guest-facing tally screen
- local persistence for the shared tally
- tablet-first layout and interaction

## Out of scope

- organizer controls
- guest-based tracking
- backend behavior
