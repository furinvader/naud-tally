# T-007: Validate the Host Workflow in a Real Usage Scenario

## Status

- open

## Goal

- Validate whether the host can use the tablet workflow quickly and confidently in realistic service conditions.

## Done when

- the host-operated pilot is tried in a real or realistic usage setting
- friction points, failure points, and missing shortcuts are recorded
- follow-up work is added to the task or decision docs

## In scope

- lightweight manual pilot testing
- observation notes
- documenting follow-up work

## Out of scope

- analytics pipelines
- backend telemetry
- broad product expansion

- [`docs/product.md`](../../product.md)

## Recommendations

- Confirm the host can open or create a guest tab quickly from room number and full name.
- Confirm the most common order can be recorded with very few taps after the guest is selected.
- Confirm product management and billing do not force the host into disruptive context switches.
- Record where offline or reconnect behavior needs clearer feedback.
- Confirm the host-operated pilot solves the real workflow well enough that public guest self-service can stay deferred.
