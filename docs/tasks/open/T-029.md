# T-029: Add Frontend Import Boundary Checks

## Status

- open

## Goal

- Enforce the intended frontend module boundaries in tooling so the architecture stays intact as the codebase grows.

## Done when

- the repo has an automated check that catches forbidden deep imports or other boundary violations in the frontend
- the allowed dependency pattern is documented clearly enough for contributors and agents
- the check runs in normal validation or CI flow

## In scope

- lint, build-time, or repo-check enforcement for frontend import boundaries
- contributor-facing documentation for the rule
- a narrow exception policy if a temporary migration escape hatch is needed

## Out of scope

- unrelated lint or formatting overhauls
- remote sync implementation
- broad refactors beyond what is needed to satisfy the new boundary checks
