# Open Task Agent Index

Read this file before opening task files under [`docs/tasks/open/`](./).

## Meaning

- This folder holds active planning work and tasks that are not done yet.
- Current-priority tasks stay in this folder; they do not use a separate status.

## Load Rules

- Open only the task that matches the current goal.
- If a task is completed, move it to the matching status folder and update [`../../tasks.md`](../../tasks.md) plus the affected status-folder indexes.

The remaining open tasks now resume with product-surface and sync work, starting from the order entry screen.
