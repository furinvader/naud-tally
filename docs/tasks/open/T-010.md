# T-010: Choose a Design Solution

## Status

- open

## Goal

- Choose a browser-accessible design solution for the project and document how it should support both humans and AI agents.

## Done when

- the primary design solution is chosen and documented
- the decision explains how designs will be shared in a browser
- the decision explains how AI agents should work with the chosen solution
- any follow-up implementation work is captured in a separate task or decision doc

## In scope

- comparing candidate design tools against the current project requirements
- deciding whether one tool or a hybrid setup is the better fit
- documenting the tradeoffs that matter for a small open-source project

## Out of scope

- producing final screens, palettes, or a full design system
- implementing Storybook or another design tool
- migrating external design assets

## Recommendations

- Preferred direction: use Penpot as the primary design tool, and keep a lightweight repo-native design spec alongside it once the design work starts.
- Penpot currently looks like the best fit when openness, browser sharing, and Codex-readability matter most.
- Penpot is browser-based and supports shareable prototype links.
- Penpot is the strongest match for the optional Codex-readable-format preference because it emphasizes open standards and design data expressed as CSS, SVG, and HTML.
- Penpot now also advertises an MCP server, which makes it a credible long-term option for AI-assisted workflows even though its practical agent workflow should still be validated.
- Penpot pricing still deserves a final check during selection: its open-source path is very attractive for small projects, but the current SaaS details are less straightforward than Figma's public pricing table.
- Strong alternative: use Figma if native AI support is the highest priority.
- Figma has the strongest first-party AI story in this comparison.
- Figma Make supports prompt-driven creation of functional prototypes, web apps, and interactive UI.
- Figma also has an MCP server and published MCP skills for AI assistants.
- Figma is easy to trial on its starter plan, but the strongest AI features are tied to paid plans and Full seats.
- Figma is a more proprietary ecosystem and a weaker match for the optional Codex-readable-file preference.
- Complementary option, not primary tool: Storybook.
- Storybook still belongs in the evaluation because it is browser-readable, open source, and very agent-friendly when the goal is implementation-adjacent UI documentation.
- Storybook is better as a complementary design-system or reference layer than as the main screen-design workspace.
- Current recommendation summary: start by evaluating Penpot and Figma as the primary options, and keep Storybook in reserve as a complementary layer rather than a replacement for a design tool.
- Decision shortcut: choose Penpot if openness and agent-readability are the top priorities; choose Figma if native AI generation is the top priority.
