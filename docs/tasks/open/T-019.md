# T-019: Make the Order Entry Screen the Default Route

## Status

- open

## Goal

- Make the order entry screen the default app surface and remove normal-user access to the public guest tally flow.

## Done when

- the order entry screen renders at the default route
- the app shell and navigation present order entry as the primary surface
- the public guest tally flow is no longer linked or reachable in normal pilot usage
- product docs and tests reflect the new host-first route structure

## In scope

- Angular route wiring
- app-shell and navigation updates built on the shared shell primitives from [`T-023`](../done/T-023.md)
- order-entry copy updates needed for routing
- frontend test coverage for the new route behavior
- product and task doc updates tied to route behavior

## Out of scope

- removing every historical guest-flow component from the codebase
- remote sync implementation
- payment processing
