# T-006: Add Host Data-Safety Tools

## Status

- open

## Goal

- Give the host explicit data-safety tools for backup, export, and guarded reset without breaking the main workflow.

## Done when

- the host can export the current working data in a simple portable format
- the host can trigger a guarded local reset with clear confirmation
- the relationship between local reset, remote recovery, and manual export is documented
- tests cover the new safety actions and destructive-action safeguards

## In scope

- export flow for current host data
- guarded reset flow
- host-facing safety messaging
- documentation updates tied to data-safety behavior

## Out of scope

- choosing the primary remote sync backend
- implementing reconnect sync
- payment processing
- simultaneous multi-tablet collaboration
