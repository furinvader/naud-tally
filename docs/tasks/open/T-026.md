# T-026: Split Tally Logic Into Guest Tabs, Catalog, and Billing History Modules

## Status

- open

## Goal

- Break the current broad tally logic into smaller capability modules that each own one durable business concept.

## Done when

- guest tabs, catalog, and billed history each have a clear owning feature area under [`../../../frontend/src/app/features/`](../../../frontend/src/app/features/)
- domain models and business rules are extracted from the current all-in-one tally implementation into those capability areas
- the new capability boundaries are visible enough that later work can target one module at a time
- feature-level tests protect the extracted business rules

## In scope

- capability extraction for guest tabs, catalog, and billed history
- domain models and business-rule ownership
- feature-owned stores or equivalent state holders for those capabilities
- documentation updates to reflect the new module map

## Out of scope

- import-boundary tooling enforcement
- remote sync implementation
- final storage-adapter migration if a temporary compatibility layer is still needed
