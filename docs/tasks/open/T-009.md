# T-009: Introduce Guest Tabs and Trust-Based Identification

## Status

- open

## Goal

- Replace the anonymous shared tally with a guest-tab model that requires room number and full name.

## Done when

- the data model can represent guests with room number, full name, and drink counts
- the UI keeps guest tabs and drinks on one screen
- a guest can create a tab once and then continue using it without re-entering identity details
- drink actions require an active selected guest and update only that guest's counts
- the shared-tablet handoff behavior intentionally reduces accidental carryover between guests
- the product, UX, and decision docs record the new model

## In scope

- guest entities
- room-number and full-name capture
- single-screen guest-tab interactions
- active guest selection and reset behavior
- updated state and test coverage

## Out of scope

- authentication
- payments
- live multi-device synchronization
- QR identification
