# T-014: Refine the Host Main Screen Layout

## Status

- open

## Goal

- Polish the host main working screen so room and name lookup, order entry, and billing context fit cleanly on a tablet without slowing service.

## Done when

- the selected guest context stays visible while the host records orders
- guest lookup or open-tab navigation and order-entry controls each stay contained within the viewport
- long guest lists and long product lists scroll inside their own panels instead of forcing page scroll
- product controls remain fast tap targets on tablet widths
- billing access is visible without overtaking the main order-entry workflow
- the UX brief and task backlog record any follow-up work

## In scope

- host main-screen layout refinement
- tablet viewport behavior
- open-tab or guest-lookup presentation changes
- order-entry panel layout refinement
- task and UX documentation updates

## Out of scope

- route changes
- remote sync implementation
- payment processing
- mobile-specific redesign beyond preserving a usable narrow layout
