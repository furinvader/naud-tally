# T-014: Refine the Order Entry Screen Layout

## Status

- open

## Goal

- Polish the order entry screen so room-first selection, guest selection, and order entry fit cleanly on a tablet without slowing service.

## Done when

- the selected guest context stays visible while the host records orders
- room selection, guest navigation, and order-entry controls each stay contained within the viewport
- long guest lists and long product lists scroll inside their own panels instead of forcing page scroll
- product controls remain fast tap targets on tablet widths
- the host-tools shortcut stays visible without overtaking the main order-entry workflow
- the UX brief and task backlog record any follow-up work

## In scope

- order entry screen layout refinement
- tablet viewport behavior
- building on the shared shell and scroll primitives from [`T-023`](../done/T-023.md) instead of reintroducing route-owned chrome
- room-list or guest-selection presentation changes
- order-entry panel layout refinement
- task and UX documentation updates

## Out of scope

- route changes
- remote sync implementation
- payment processing
- mobile-specific redesign beyond preserving a usable narrow layout
