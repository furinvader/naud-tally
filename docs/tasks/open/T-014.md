# T-014: Refine the Public Guest Screen Layout

## Status

- open

## Goal

- Polish the shared-tablet guest screen so the main public and personal areas feel cleaner, calmer, and better contained within the viewport.

## Done when

- active guest cards fill the full width of their list container
- the left guest-list area and the right personal-panel area each fill the remaining tablet or desktop screen height
- long guest lists and long selected-guest content scroll inside their own panels instead of forcing page scroll
- the empty personal-panel placeholder shows its explanatory copy before a merged summary block
- the empty-state summary is rendered as one flatter combined surface instead of two separate stat cards
- drink counters use a more compact row layout with the recorded count aligned beside the drink name
- the selected-guest inactivity hint shows an animated fill and countdown ring during the existing 90-second timeout
- the UX brief and task backlog record the follow-up work

## In scope

- shared-tablet layout refinement for the public tally view
- active guest list presentation changes
- selected-guest panel layout refinement
- compact drink-counter card presentation
- component-local timeout-progress UI state
- task and UX documentation updates

## Out of scope

- route changes
- guest data-model or persistence changes
- organizer workflows
- pricing or billing logic
- mobile-specific redesign beyond preserving a usable narrow layout
