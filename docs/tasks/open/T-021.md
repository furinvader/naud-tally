# T-021: Choose the Remote Backup and Sync Approach

## Status

- open

## Goal

- Choose and document the remote backup or sync approach, data model, conflict rules, and recovery flow for the host-operated pilot.

## Done when

- the chosen remote approach is recorded in the docs with rationale
- the host setup and authentication flow are defined clearly enough to implement
- the remote data model, sync unit, conflict rule, and recovery flow are documented
- product, decision, and task docs reflect the chosen direction
- implementation acceptance criteria exist for the follow-up sync task

## In scope

- research synthesis
- product and decision updates
- recovery and reconnect-flow definition
- acceptance criteria for implementation

## Out of scope

- frontend or backend implementation
- payment processing
- simultaneous multi-tablet collaboration policy beyond what the chosen sync model requires

## Recommendations

- Prefer a Firestore-style app database for primary remote sync and recovery.
- Keep Google Sheets as a backup, export, or reporting target rather than the primary source of truth.
- Preserve local-first writes even after remote sync is introduced.
