# T-018: Design the Order Entry Screen

## Status

- open

## Goal

- Define the order entry screen for rapid room-first guest selection and order entry on a tablet, with host tools handling room setup, product management, and billing for now.

## Done when

- the primary order entry screen model is documented clearly enough to implement
- the room-selection and guest lookup or creation path is defined around `select room -> select guest -> orders`
- the placement of order-entry and host-tool actions is decided
- the tablet information hierarchy and key interaction states are captured in UX or design docs
- follow-up implementation gaps are recorded in the backlog if needed

## In scope

- order entry screen interaction model
- tablet layout and hierarchy
- room-selection, guest-selection, and selected-guest workflow design
- design or UX documentation updates

## Out of scope

- route implementation
- a future overview screen that may later replace order entry as the landing surface
- PWA implementation
- remote sync implementation
- payment processing
