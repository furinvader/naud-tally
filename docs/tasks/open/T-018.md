# T-018: Design the Host Main Working Screen

## Status

- open

## Goal

- Define the host main working screen for rapid room-and-name lookup, order entry, product access, and billing on a tablet.

## Done when

- the primary host screen model is documented clearly enough to implement
- the guest lookup or creation path is defined around `room number -> full name`
- the placement of order-entry, product-management, and billing actions is decided
- the tablet information hierarchy and key interaction states are captured in UX or design docs
- follow-up implementation gaps are recorded in the backlog if needed

## In scope

- host main-screen interaction model
- tablet layout and hierarchy
- open-tab and selected-guest workflow design
- design or UX documentation updates

## Out of scope

- route implementation
- PWA implementation
- remote sync implementation
- payment processing
