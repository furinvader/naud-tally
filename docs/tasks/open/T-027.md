# T-027: Introduce Repository Adapters for Local Persistence

## Status

- open

## Goal

- Move local persistence behind explicit repository or data-adapter boundaries so capability code stops talking to browser storage APIs directly.

## Done when

- capability state reads and writes through repositories or equivalent data adapters
- direct browser-storage access is removed from route composition code and business-rule files
- current local persistence behavior still works after the refactor
- the persistence seam is clear enough for later sync and recovery work to build on it

## In scope

- repository interfaces or equivalent data-adapter seams
- local serialization and hydration logic
- compatibility-preserving refactors of current local persistence
- tests for local loading and saving behavior where the owning feature needs them

## Out of scope

- final remote backend selection
- remote sync implementation
- user-facing backup flows beyond what is needed to preserve current local behavior
