# T-024: Create the Host Workspace Composition Root

## Status

- open

## Goal

- Introduce a dedicated host-workspace feature that becomes the route-level composition root for the pilot's main screen.

## Done when

- a new host-workspace feature area exists under [`../../../frontend/src/app/features/`](../../../frontend/src/app/features/)
- the host-workspace feature has a clear owning component and any needed route-level facade or store
- the host-workspace feature is documented as the intended composition root in the frontend structure
- the current implementation still works while the migration continues

## In scope

- route-level host workspace feature structure
- composition responsibilities for the main host screen
- initial wiring needed to let later refactors move into that feature cleanly
- documentation updates that keep the new ownership obvious

## Out of scope

- final host-screen visual redesign
- remote sync implementation
- large persistence refactors beyond what is needed to introduce the composition root
