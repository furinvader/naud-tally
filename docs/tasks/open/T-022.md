# T-022: Implement Remote Sync and Recovery

## Status

- open

## Goal

- Add remote sync, reconnect behavior, and replacement-device recovery on top of the local-first host workflow.

## Done when

- local changes for guest tabs, billed history, and product catalog sync to the chosen remote store
- the app can restore prior state on a reinstalled or replacement device
- reconnect behavior safely resumes deferred sync after offline use
- failure handling and sync-state messaging are clear enough for the pilot
- tests and build pass

## In scope

- sync adapter or service layer
- local-to-remote write flow
- restore flow for remote recovery
- reconnect behavior and basic sync-state UI
- frontend test coverage
- product and task doc updates tied to sync behavior

## Out of scope

- simultaneous live collaboration across multiple active host devices
- payment processing
- public guest self-service access
