# T-020: Add PWA Installability and Offline Shell Support

## Status

- open

## Goal

- Make the host-operated app installable as a PWA and keep the core shell usable offline after the first successful load.

## Done when

- the app exposes installable PWA metadata
- the order entry route and core shell assets are available offline after initial load
- offline behavior is documented clearly for the pilot
- tests and build checks cover the new PWA wiring where practical

## In scope

- Angular PWA or service-worker setup
- install metadata and manifest work
- offline shell caching for the core order-entry experience
- product and task doc updates tied to offline shell behavior

## Out of scope

- remote sync backend selection
- replacement-device recovery by itself
- simultaneous multi-tablet live collaboration
- payment processing
