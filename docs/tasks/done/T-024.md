# T-024: Create the Order Entry Composition Root

## Status

- done

## Goal

- Introduce a dedicated order-entry feature that becomes the route-level composition root for the pilot's current main screen.

## Done when

- a new order-entry feature area exists under [`../../../frontend/src/app/features/`](../../../frontend/src/app/features/)
- the order-entry feature has a clear owning component and any needed route-level facade or store
- the order-entry feature is documented as the intended composition root in the frontend structure
- the current implementation still works while the migration continues

## In scope

- route-level order-entry feature structure
- composition responsibilities for the current main order entry screen
- initial wiring needed to let later refactors move into that feature cleanly
- documentation updates that keep the new ownership obvious

## Out of scope

- final order entry screen visual redesign
- remote sync implementation
- large persistence refactors beyond what is needed to introduce the composition root
