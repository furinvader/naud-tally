# T-016: Add a Host Admin Route

## Status

- done

## Goal

- Give the organizer a dedicated route where drinks can be managed and guest tabs can be billed without disrupting the public guest tally flow.

## Done when

- the app has a separate host route for organizer-only tasks
- the host can add drinks to the live catalog
- the host can remove unused drinks from the live guest catalog
- the host can bill a guest tab and move it out of the active guest list
- billed guest history and catalog changes persist locally across reloads
- product and task docs reflect the new scope
- frontend tests and build pass

## In scope

- Angular route wiring for a separate host screen
- persisted drink-catalog management in shared tally state
- host-side guest billing and billed history
- public guest-flow updates needed to respect the live catalog
- frontend test coverage for host route behavior
- product and task doc updates

## Out of scope

- authentication or role-based access control
- payment processing or external billing integrations
- export or reset tooling
- inventory tracking
- multi-device synchronization

## Outcome

- added a dedicated `/host` route with a standalone host admin screen
- moved the drink catalog into persisted shared state so the host can add and remove drinks locally
- added host-side billing that converts open guest tabs into a billed-history view and removes them from the public guest list
- kept the guest route focused on quick tallying while exposing a small route link to reach host tools
- expanded frontend tests to cover route rendering, host admin interactions, and the new store behavior
