# T-018: Design the Order Entry Screen

## Status

- done

## Goal

- Define the order entry screen as a focused stepper for rapid room-first guest selection, ordered-drinks-first entry, and lightweight inline guest billing on a tablet, with host tools handling room setup, product management, and broader admin review.

## Done when

- the primary order entry screen model is documented clearly enough to implement
- the room-selection and guest lookup or creation path is defined around `select room -> select guest -> orders`
- the screen model keeps one workflow step visible at a time while leaving completed steps tappable for quick corrections
- room and guest selection are defined as touch-friendly grid presentations
- the drinks step is defined around ordered drinks only, with a lightweight add-drink picker for bringing in new drink types
- the drinks step includes a lightweight billing-review modal for confirming and closing the current tab
- the placement of order-entry and host-tool actions is decided
- the tablet information hierarchy and key interaction states are captured in UX or design docs
- follow-up implementation gaps are recorded in the backlog if needed

## In scope

- order entry screen interaction model
- stepper navigation and correction behavior
- tablet layout and hierarchy
- room-selection, guest-selection, and selected-guest workflow design
- ordered-drinks-first drinks-step design
- inline billing-review design
- room-grid and guest-grid presentation
- design or UX documentation updates

## Out of scope

- route implementation
- a future overview screen that may later replace order entry as the landing surface
- PWA implementation
- remote sync implementation
- payment processing
