# T-028: Expose Feature Public APIs and Remove Cross-Feature Internal Imports

## Status

- done

## Goal

- Make feature boundaries explicit by introducing small public APIs or facades and migrating feature consumers away from deep imports.

## Done when

- each larger capability feature exposes an intentional public API or facade
- route features and supporting screens consume those public APIs instead of deep-importing another feature's internal files
- existing cross-feature deep imports that break the intended boundaries are removed
- the frontend docs reflect the allowed dependency pattern

## In scope

- public API files or facades
- refactoring feature consumers to use those APIs
- documentation updates for allowed dependency patterns

## Out of scope

- tooling enforcement for import boundaries
- broad visual redesign work
- remote sync implementation
