# T-011: Document the Penpot Handoff Workflow

## Status

- done

## Goal

- Make Penpot usable as a Codex-first design workspace, with the smallest possible amount of browser-only setup.

## Done when

- the repo has a repeatable way to start Penpot MCP locally
- the repo has a repeatable project-scoped Codex config with Penpot prewired
- a repo-native markdown pattern exists for Penpot design handoff notes
- the pattern says where committed design artifacts should live in the repo
- the pattern lists the minimum states, annotations, and implementation notes an agent should receive
- the one unavoidable browser-side step is documented clearly instead of being treated as hidden setup
- a simple example Penpot file is created and a repo-tracked design export is recorded in the repo

## In scope

- documenting the Codex-side MCP setup
- documenting the minimum fields for a Penpot handoff brief
- deciding where to keep committed design artifacts and optional Penpot links in the repo
- creating a simple Penpot example brief that Codex can use once connected
- documenting how AI agents should use the handoff material during implementation

## Out of scope

- polished final screens in Penpot
- building Storybook
- changing the product scope
