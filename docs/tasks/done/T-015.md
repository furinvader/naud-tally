# T-015: Simplify Selected-Guest Drink Entry

## Status

- done

## Goal

- Reduce clutter in the selected-guest tally surface by separating already-recorded drinks from drinks that have not been used yet.

## Done when

- the selected guest panel shows a focused `Your drinks` section for drinks with count above zero
- the panel shows a separate `Add a drink` section for zero-count drinks
- tapping a drink in `Add a drink` records the first drink immediately
- decrementing a drink to zero removes it from `Your drinks` and returns it to `Add a drink`
- active drink ordering stays stable while a guest is working, then re-sorts from current counts when the panel is opened again
- product, UX, and task docs reflect the new interaction model
- frontend tests and build pass

## In scope

- selected guest store-derived view-model changes
- non-persisted selected-panel ordering state
- selected guest panel UI refinement
- available-drink presentation component
- frontend test updates
- product and UX doc updates

## Out of scope

- changes to the persisted guest-tab schema
- organizer workflows
- billing or subtotal logic
- route changes
- multi-device synchronization

## Outcome

- replaced the selected guest’s single full-catalog tally list with split `Your drinks` and `Add a drink` sections
- kept `DrinkCounts` as the persisted source of truth while adding transient state to preserve stable active-drink ordering during an open session
- introduced a compact available-drink grid that records the first drink with one tap
- removed dead selected-panel timeout-progress wiring that no longer had any visible UI
- expanded frontend tests to cover add-first entry, zero-count removal, stable ordering, and the new selected-panel layout
