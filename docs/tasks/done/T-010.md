# T-010: Choose a Design Solution

## Status

- done

## Goal

- Choose a browser-accessible design solution for the project and document how it should support both humans and AI agents.

## Done when

- the primary design solution is chosen and documented
- the decision explains how designs will be shared in a browser
- the decision explains how AI agents should work with the chosen solution
- any follow-up implementation work is captured in a separate task or decision doc

## In scope

- comparing candidate design tools against the current project requirements
- deciding whether one tool or a hybrid setup is the better fit
- documenting the tradeoffs that matter for a small open-source project

## Out of scope

- producing final screens, palettes, or a full design system
- implementing Storybook or another design tool
- migrating external design assets

## Recommendations

- Final recommendation: choose Penpot as the primary design workspace for this project.
- Keep Figma as the fallback option if Penpot blocks progress or if prompt-driven generation becomes the deciding factor in real usage.
- Storybook remains a likely later addition for component behavior, implementation-adjacent documentation, and showing project maturity, but it should not replace the primary design workspace.
- If Penpot and Figma both disappoint in practice, the next tools worth checking are Uizard and Pixso.
- Complementary option, not primary tool: Storybook.
- Storybook still belongs in the evaluation because it is browser-readable, open source, and very agent-friendly when the goal is implementation-adjacent UI documentation.
- Storybook is better as a complementary design-system or reference layer than as the main screen-design workspace.
- Current recommendation summary: use Penpot by default, keep Storybook in reserve as a complementary layer, keep Figma as the main fallback, and treat Uizard and Pixso as lower-priority explorations rather than equal first-pass candidates.
