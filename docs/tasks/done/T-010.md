## Recommendations

- Final recommendation: choose Penpot as the primary design workspace for this project.
- Keep Figma as the fallback option if Penpot blocks progress or if prompt-driven generation becomes the deciding factor in real usage.
- Storybook remains a likely later addition for component behavior, implementation-adjacent documentation, and showing project maturity, but it should not replace the primary design workspace.
- If Penpot and Figma both disappoint in practice, the next tools worth checking are Uizard and Pixso.
- Complementary option, not primary tool: Storybook.
- Storybook still belongs in the evaluation because it is browser-readable, open source, and very agent-friendly when the goal is implementation-adjacent UI documentation.
- Storybook is better as a complementary design-system or reference layer than as the main screen-design workspace.
- Current recommendation summary: use Penpot by default, keep Storybook in reserve as a complementary layer, keep Figma as the main fallback, and treat Uizard and Pixso as lower-priority explorations rather than equal first-pass candidates.

## Research Notes

- Research snapshot date: April 1, 2026.
- Penpot strengths:
  - browser-based with shareable prototype links
  - strongest match for the optional Codex-readable-format preference because it emphasizes open standards and code-oriented inspect output
  - supports inspect/code workflows around CSS, SVG, and HTML
  - open-source and self-hostable, which lowers long-term lock-in risk
  - now has an official MCP server project, which makes it more credible for future agent workflows
- Penpot open questions:
  - the hosted pricing story is less straightforward than Figma's public pricing page
  - the practical quality of its AI-agent workflow still needs validation in real use
- Figma strengths:
  - strongest first-party AI story in this comparison
  - Figma Make supports prompt-driven creation of functional prototypes, web apps, and interactive UI
  - sharing and browser access are mature and widely understood
  - Figma now documents both an MCP server and custom MCP connectors for Make
  - its pricing page is clearer than Penpot's for small-team evaluation
- Figma open questions:
  - AI-credit usage is difficult to predict before real usage
  - the strongest AI workflows are still more attractive on paid plans and Full seats
  - it is a more proprietary ecosystem and a weaker match for the optional readable-file preference
- Storybook note:
  - Storybook is likely valuable later to document component behavior and professionalism, but it should be treated as a companion layer after the primary design tool is chosen
- Uizard note:
  - strongest AI-first fallback candidate found so far
  - browser-based, shareable, free to start, and explicitly offers CSS and React handoff on paid plans
  - weaker than Penpot and Figma on openness and long-term agent integration
- Pixso note:
  - most interesting dark horse after the primary shortlist
  - browser-based collaboration with an MCP guide, but it needs more trust and workflow validation before being recommended above Penpot or Figma
- Framer note:
  - capable and AI-assisted, but it appears more aligned with designing and publishing websites than with maintaining the main product-design source for this app

## Manual Evaluation Checklist

- Can a 3-screen flow be created and shared in the browser quickly?
- Can non-editors review the design cleanly without setup friction?
- Can the tool describe layouts, spacing, colors, and interaction states clearly enough for an AI agent to implement them accurately?
- Can the output be exported or inspected in a way that feels durable enough to avoid tool lock-in?
- Does the free tier remain useful for a very small project?
- If the free tier is not enough, does the paid path still feel reasonable for one or two humans?

## Outcome

- kept Figma as the fallback when prompt-driven generation or Penpot friction outweighs the benefits of openness and inspectability
- kept Storybook as a later complementary layer for implementation-adjacent component documentation rather than the main design workspace
