# T-023: Introduce Shared UI Shell and Scroll Foundations

## Status

- done

## Goal

- Extract reusable tablet shell, app-bar, and scroll foundations from the current routes so shared UI stops living inside feature-owned views.

## Done when

- shared UI primitives exist under [`frontend/src/app/ui/`](../../../frontend/src/app/ui/)
- the current tally and host routes compose shared shell and app-bar primitives instead of duplicating toolbar and viewport code
- scroll-shadow behavior lives in shared UI and current panel-local scroll areas use the shared scroll-region wrapper
- frontend docs describe the new `src/app/ui/` convention and the host UX brief records the tablet scroll behavior
- frontend tests and build pass

## In scope

- shared page-shell, app-bar, scroll-region, and scroll-shadow primitives
- route template refactors for the current tally and host views
- global viewport behavior updates for tablet layouts
- task, UX, and frontend documentation updates

## Out of scope

- redesigning the host workflow or route hierarchy
- extracting every repeated card or content block into shared UI
- remote sync, billing, or product-management feature changes

## Outcome

- added [`frontend/src/app/ui/`](../../../frontend/src/app/ui/) for cross-feature page-shell, app-bar, scroll-region, and scroll-shadow primitives
- moved the tally and host routes onto shared shell and app-bar composition without centralizing route chrome in the root app component
- moved scroll-shadow behavior out of the tally feature and reused it through the shared scroll-region wrapper
- tightened the global viewport contract so tablet layouts avoid browser-page scrolling on wider screens while keeping a narrow-width fallback
- documented the shared-UI convention and the host UX expectation that anchored chrome and owned scroll regions should be preferred on tablet screens
