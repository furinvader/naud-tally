# T-009: Introduce Guest-Based Tracking

## Status

- open

## Goal

- Move beyond a single shared tally and attach counts to guests.

## Done when

- the data model can represent guests and their drink counts
- the UI supports assigning counts within a guest context
- the shared tally flow is intentionally evolved rather than left half-preserved
- the product and decision docs record the new model

## In scope

- guest entities
- guest-based tally interactions
- updated state and test coverage

## Out of scope

- authentication
- payments
- live multi-device synchronization
