# Timeless Wording

Use this reference to catch wording that becomes stale quickly.

This guidance is inspired by Google's timeless-documentation principle: prefer wording that remains true without depending on the reader knowing when the page was last updated.

## High-Risk Phrases

Flag these when they are not anchored to a clear date or version:

- `currently`
- `now`
- `latest`
- `new`
- `soon`
- `at present`
- `still`
- `next`
- `does not yet`
- `for now`

## Better Patterns

- replace relative time with a durable statement
- use an exact date when the timing itself matters
- say what is true, not what recently changed
- say what owns the behavior, not what changed in chat or in memory

## Example Rewrites

- `The next work is to make X the default route.` -> `X is the default route.` or `T-020 is the current priority.`
- `The app currently uses...` -> `The app uses...`
- `This is still deferred.` -> `This remains deferred future work.` or remove the sentence if the status is already clear
